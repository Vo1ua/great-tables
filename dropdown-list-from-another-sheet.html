How to Create a Drop-Down List in Google Sheets from Another Sheet

Drop-down lists keep data entry clean and consistent — instead of people typing "USA," "U.S.A.," and "United States" into the same column, they pick one option from a fixed list. Things get slightly trickier when the list of options lives on a different tab than the cell you want the dropdown in. This guide walks through exactly how to pull that off, from the basic range reference to a fully dynamic setup with named ranges.

The Basic Method: Range Reference

Let's say you have a tab called Data that lists valid options in column A, and you want a dropdown on your Form tab that pulls from it.

  1. Go to the Form tab and select the cell (or range of cells) where you want the dropdown to appear.
  2. Open the menu: Data → Data validation.
  3. Click Add rule.
  4. Under Criteria, choose Dropdown (from a range).
  5. Click the small grid icon and select the range on your Data tab — for example Data!A2:A20.
  6. Click Done.
Your Form tab now shows a dropdown chip. Clicking it displays the exact list of options stored on the Data tab — and if you edit that list later, the dropdown updates automatically.

Making the List Grow Automatically

A fixed range like Data!A2:A20 works, but it won't include new items you add below row 20. To avoid manually expanding the range every time, use an open-ended reference instead:

Data!A2:A

This tells Google Sheets to include everything from row 2 down to the last filled row in column A — no matter how many items you add later.

Using a Named Range Instead

If you reuse the same list in several dropdowns across your spreadsheet, a named range keeps things easier to manage than repeating the same tab-and-cell reference everywhere.

  1. Select the range on your Data tab (for example A2:A).
  2. Go to Data → Named ranges.
  3. Give it a clear name, like CategoryList, and click Done.
  4. Back in Data validation on your Form tab, choose Dropdown (from a range) and type =CategoryList instead of the cell reference.

Now, if you ever move the source data to a different tab, you only need to update the named range once — every dropdown using it will update automatically.

Dependent Dropdowns (Optional, More Advanced)

Sometimes one dropdown should depend on another — for example, choosing a "Category" first should limit the "Item" dropdown to only items in that category. This usually requires combining named ranges with the INDIRECT function, so the Data Validation range changes based on another cell's value. It's a more advanced setup worth its own separate tutorial — but the two techniques above are the foundation it's built on.

Common Issues

Dropdown Shows No Options

This usually means the range reference points to empty cells, or the sheet name in the reference doesn't match the actual tab name exactly. Double-check both.

List Includes Blank Options

If your range includes empty rows at the end (like A2:A50 when you only have 10 items), the dropdown will show blank entries too. Switch to an open-ended range like A2:A to avoid this.

Bonus: Pulling the List from a Different Spreadsheet File

Everything above assumes both tabs are in the same spreadsheet file. If your list of options lives in a completely separate Google Sheets file, you'll need to bring the data in first using the IMPORTRANGE function, then build your dropdown from that imported range. We covered the basics of cross-file references in an earlier article on referencing cells between tabs.

Once you're comfortable pulling dropdown options from another tab, you can separate your raw data from the parts of the spreadsheet people actually interact with — keeping things cleaner, easier to update, and less error-prone for anyone entering data.

Frequently Asked Questions

Will the dropdown update if I add new items to the source list?

Only if you use an open-ended range like Data!A2:A instead of a fixed range like Data!A2:A20. Open-ended ranges automatically include new rows.

Can I use a dropdown list from a different spreadsheet file?

Not directly. You'll first need to bring the data in using the IMPORTRANGE function, then build the dropdown from that imported range.

Why does my dropdown show blank options?

This usually happens when the selected range includes empty rows at the end. Switch to an open-ended range like A2:A to fix it.
