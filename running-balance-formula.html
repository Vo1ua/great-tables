Google Sheets Formula to Calculate a Running Balance

A running balance shows your account balance after every single transaction, not just a final total at the bottom. It's the difference between "here's what you spent this month" and "here's exactly how much you had left after each purchase" — and it's one of the most useful things you can add to any ledger, budget, or expense tracker.

Setting Up Your Columns

A typical running balance setup uses four columns: date, income (money in), expense (money out), and the balance itself.

DateIncomeExpenseBalance
7/120002000
7/3851915
7/105002415

The first row's balance is just your starting amount. Every row after that adds the new income, subtracts the new expense, and carries forward from the previous balance.

Method 1: The Drag-Down Formula (Simplest)

In the first data row (row 2), set the starting balance directly — just type the number. In the second data row (row 3), enter:

=D2+B3-C3

This takes the previous row's balance (D2), adds this row's income (B3), and subtracts this row's expense (C3).

  1. Click the cell with this formula.
  2. Double-click the small blue square (fill handle) in its bottom-right corner, or drag it down manually.
Google Sheets automatically adjusts the row numbers as it copies down, so row 4 becomes =D3+B4-C4, row 5 becomes =D4+B5-C5, and so on.

The one drawback: you have to remember to drag the formula down every time you add a new transaction. If you forget, the new row simply won't have a balance.

Method 2: ARRAYFORMULA (Fully Automatic)

If you'd rather never drag a formula down again, a single ARRAYFORMULA in the very top cell can calculate the entire column at once, automatically extending as new rows are added:

=ARRAYFORMULA(IF(B2:B="", "", SUM(B$2:B2)-SUM(C$2:C2)))

Here's what each part is doing:

  • IF(B2:B="", "", ...) — leaves the cell blank if there's no income entered on that row, so the formula doesn't spill into unused rows
  • SUM(B$2:B2) — adds up all income from row 2 down to the current row, using a mix of an absolute and relative reference so the starting point never moves
  • SUM(C$2:C2) — does the same for expenses

Enter this once in cell D2, and the whole column fills in on its own — including any new rows you add later, with no copying or dragging required.

Adding a Starting Balance

If you're tracking an existing account rather than starting from zero, add your starting balance to either version of the formula:

=$G$1+SUM(B$2:B2)-SUM(C$2:C2)

Where G1 holds your starting balance in a fixed cell, referenced with dollar signs so it never changes as the formula copies down.

Common Issues

Balance Shows 0 in Rows Below My Actual Data

This happens when an ARRAYFORMULA version doesn't check for blank rows first. Make sure the formula includes an IF(B2:B="", "", ...) check, as shown above, so it stops calculating once it reaches empty rows.

Balance Doesn't Match My Bank Statement

Double-check that every transaction is entered as either income or an expense, not both, and that no rows were skipped. A single missing entry throws off every balance after it.

Frequently Asked Questions

What's the difference between a running balance and a regular SUM?

SUM gives one final total for a whole range. A running balance shows the cumulative total after every individual row, so you can see the balance at any point in time, not just at the end.

Do I need to know ARRAYFORMULA to build a running balance?

No. The simple drag-down method works perfectly well for most spreadsheets. ARRAYFORMULA is only useful if you want the balance to fill in automatically without dragging the formula down yourself.

Why does my running balance show 0 in empty rows below my data?

This happens with ARRAYFORMULA versions that don't check for blank rows. Wrapping the formula in an IF that checks whether the row has data prevents the balance from repeating into empty rows.
