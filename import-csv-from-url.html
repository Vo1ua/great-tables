Google Sheets: How to Import CSV Data from a URL

If a CSV file is published somewhere online — a government open-data portal, an API endpoint, a public report — you don't need to download it and re-upload it manually. Google Sheets has a built-in function that pulls the data straight from the URL, and it's a single line.

The Formula

In any empty cell, enter:

=IMPORTDATA("https://example.com/data.csv")

The data appears starting in the cell where you entered the formula and automatically spills into the surrounding cells — one row and column per row and column in the source file.

Importing TSV Files

IMPORTDATA works with both CSV (comma-separated) and TSV (tab-separated) files — no separate function or setting is needed. Google Sheets detects the delimiter automatically based on the file's structure.

Requirements for the Source URL

  • The URL must be publicly accessible — no login, password, or API key
  • The file must actually be in CSV or TSV format at that address, not an HTML page linking to a download
  • The total data can't exceed 50,000 cells

Keeping the Data Fresh

Once entered, IMPORTDATA automatically re-checks the source URL periodically and whenever the spreadsheet is opened, so if the file at that address updates, your sheet updates with it — no manual re-import needed.

If you need to force an immediate refresh without waiting, a common trick is to re-enter the exact same formula (delete it and retype it, or use Ctrl+Z followed by Ctrl+Y), which prompts Google Sheets to re-fetch the data right away.

Common Issues

#N/A "Could Not Fetch URL"

This is the most common error, and it almost always means the URL itself is the problem — not IMPORTDATA. Double-check for typos in the file extension (a common one is .cs instead of .csv), and confirm the file loads correctly when you paste the URL directly into a browser.

Only Part of the Data Loads

If the source file has more columns than exist in your sheet near the formula, IMPORTDATA won't automatically create new columns to fit everything — it will show an error instead. Make sure there's enough empty space to the right of the formula before running it.

Import Fails on Private or Authenticated URLs

IMPORTDATA has no way to send login credentials or API keys. If the file requires authentication, it needs to be made publicly accessible first, or pulled in through a different method such as Apps Script with proper authentication handling.

Frequently Asked Questions

Does IMPORTDATA update automatically if the source file changes?

Yes, IMPORTDATA re-checks the source URL periodically and on file open, so the imported data stays in sync with changes at that URL without needing to be re-entered.

Can IMPORTDATA import from a private or password-protected file?

No. IMPORTDATA can only access files that are publicly available at a direct URL, with no login or authentication required.

Is there a size limit for IMPORTDATA?

Yes. IMPORTDATA can import a maximum of 50,000 cells at a time, and very large source files may load slowly or fail.
